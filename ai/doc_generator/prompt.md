# Module Documentation Generator

Your task is to take a code description and documentation from child modules to generate a cohesive, high-level overview for a RigBuilder module.

## Instructions

1. **Synthesize Information**: Create a unified description that captures the essence of the module's behavior and its sub-components.
2. **Focus on Purpose**: Highlight and emphasize the high-level purpose and functional role of the module.
3. **Maintain Tone**: Use a technical, clear, and direct tone.
4. **Structure**:
   - Start with a high-level summary paragraph (1-3 sentences).
   - Follow with a list of key points or features.
   - End with potential use cases.

## Output Format

Example of output you should generate:

```markdown
## Summary
The module creates bipedal rig, containing arms, legs, spine, and head. 

## Features
- Arms and legs have IK/FK switch, dynamic parents and stretch.
- Spine has IK/FK switch and stretch.
- Head has IK/FK switch and stretch.

## Use cases
- Create a bipedal rig.
```
